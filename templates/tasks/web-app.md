## 界面与交互要求

- 第一屏应直接呈现可使用的核心体验，不要只做营销页。
- 页面应适配桌面和移动端。
- 表单、按钮、加载、空状态、错误提示等状态应完整。
- UI 应与项目主题匹配，避免只使用单一色系或过度装饰。
- 如果使用图标，优先使用项目已有图标库。

## 技术要求

- 优先使用项目已有技术栈。
- 不要引入不必要的复杂依赖。
- 保持组件、状态和样式结构清晰。
- 如需新增脚本，请同步更新运行说明。

## 验收标准

- 应用可以启动或构建。
- 核心页面和核心交互可用。
- 移动端和桌面端没有明显布局错乱。
- 关键功能经过验证。

## 验证要求

优先运行项目已有命令：

```bash
npm run lint
npm run build
npm test
```

如果项目没有这些命令，请执行最小 smoke check 并说明验证方式。

## 输出要求

完成后请说明：

- 入口页面或运行方式。
- 新增和修改的主要文件。
- 运行过的验证命令和结果。
- 剩余风险或下一步建议。
