## 检查清单

- README 是否讲清楚核心价值和快速开始。
- CHANGELOG 是否记录本次变化。
- Release notes 是否准确、简洁。
- 安装或配置说明是否仍然正确。
- CLI help、文档命令、示例命令是否一致。
- 是否存在本地私有路径、密钥、临时文件或不可公开内容。

## 验收标准

- 发布材料准确描述当前功能。
- 用户能根据 README 跑通最短路径。
- CI 或本地最小验证通过。
- 没有公开安全风险。

## 验证要求

优先运行：

```bash
node --check cli/vibe-cli/bin/vibe
node --check cli/agent-runner/bin/agent-runner
git diff --check
```

如项目有 CI、build 或 test 命令，也应运行相关最小检查。

## 输出要求

完成后请说明：

- 发布材料更新了什么。
- 运行了哪些验证。
- 是否发现公开风险。
- 建议的提交信息或发布标题。
